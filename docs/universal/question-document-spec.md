# 質問ドキュメント仕様

## 目的

Claude Code とユーザーのやり取りが以下のような状況になる場合に使用する：

- 複数の質問が相互に依存している（Q2 の答えが Q1 の答えによって変わる）
- 決定内容を記録として残す必要がある
- 複数の前提条件を整理してから判断が必要
- アーキテクチャや方針に関わる重要な決定
- 新規プロジェクトの初期設定

一問一答で済む単純な質問には使用しない。

## ファイルの配置

```
.claude/questions/YYYYMMDD-<topic>.md
```

例：

```
.claude/questions/20260402-new-project.md
.claude/questions/20260402-error-fix.md
.claude/questions/20260402-architecture-change.md
```

## ファイルの命名規則

- `YYYYMMDD`: 作成日
- `topic`: 内容を表す短い英語のキーワード
  - `new-project` / `error-fix` / `architecture` / `deployment` / `refactor` など

## ファイルのフォーマット

以下のテンプレートに従うこと：

```markdown

- 作成日時: YYYY-MM-DD HH:MM

（なぜこの質問ドキュメントを作成したか。
```

## ワークフロー

### Claude Code の動作フロー

```
複雑な判断が必要な状況を検出
  ↓
質問ドキュメントを作成
  .claude/questions/YYYYMMDD-topic.md
  ↓
ユーザーに通知：
  「質問ドキュメントを作成しました。
   .claude/questions/YYYYMMDD-topic.md
   を Cursor で開いて回答を記入してください。
   記入が完了したら『回答を読んで実行して』と伝えてください。」
  ↓
ユーザーが Cursor で回答を記入
  ↓
「回答を読んで実行して」とトリガー
  ↓
Claude Code がドキュメントを読み込む
  ↓
回答の解釈を「Claude Code が実行するアクション」欄に記入
  ↓
ユーザーに確認：
  「以下の内容で実行します。よろしいですか？
  （実行内容のサマリー）」
  ↓
YES → 実行開始
NO  → 再度質問ドキュメントを更新してユーザーに修正を依頼
  ↓
実行完了
  ↓
質問ドキュメントのステータスを「実行済み」に更新
実行結果を「実行結果」欄に記入
コミット
```

### ユーザーの動作フロー

```
Claude Code から通知を受ける
  ↓
Cursor で質問ドキュメントを開く
  ↓
背景・前提条件を読む
  ↓
各質問に回答を記入する
（依存関係に注意して順番に回答する）
  ↓
追加コメントがあれば記入する
  ↓
ファイルを保存する
  ↓
Claude Code に「回答を読んで実行して」と伝える
```

## 質問ドキュメントを使うべき判断基準

以下のいずれかに該当する場合は質問ドキュメントを作成する：

| 条件 | 例 |
|---|---|
| 質問数が 3 つ以上 | 新規プロジェクトの初期設定 |
| 質問間に依存関係がある | Q2 の答えが Q1 に依存する |
| 決定内容を記録として残したい | アーキテクチャの選定 |
| 複数の選択肢に前提条件がある | デプロイ戦略の選択 |
| 回答に調査・検討が必要 | パフォーマンス要件の確認 |
| 複数人で合意が必要 | チームの技術選定 |

以下の場合は一問一答で対応する：

| 条件 | 例 |
|---|---|
| 質問が 1〜2 個 | 「Python と Node.js どちらですか？」 |
| 即答できる | 「リージョンはどこですか？」 |
| 記録不要 | 「このファイル名でいいですか？」 |

## 質問ドキュメントの保管

- `.claude/questions/` に蓄積していく
- 実行済みのものも削除せず保管する（意思決定の記録として活用できるため）
- `.gitignore` には追加しない（チームで共有できるようにする）
