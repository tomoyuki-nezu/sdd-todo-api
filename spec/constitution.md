# プロジェクト憲法

## Green Zone（AI が自律判断してよい）
- src/ 以下のコードの生成・修正
- tests/ 以下のテストコードの生成・修正
- template.yaml の生成・更新
- docs/ 以下のドキュメント生成

## Yellow Zone（Human の承認が必要）
- 本番環境へのデプロイ
- API インターフェースの破壊的変更
- DynamoDB テーブル定義の変更
- IAM ポリシーの変更

## Red Zone（絶対禁止）
- 本番データへの直接アクセス
- シークレット・認証情報のコード内記述
- この constitution.md の変更
- Human Gate の削除・バイパス

## エスカレーション条件
以下の場合は処理を中断し Human に通知すること：
- 仕様書の解釈が複数に分かれる場合
- テスト修正ループが 3 回を超えた場合
- Red Zone の操作が必要と判断された場合
