# Test Workflow

## テスト実行の標準手順

### 1. ユニットテストの実行

```bash
pytest tests/unit/ -v
```

実行後に以下を確認：
- 全テストのパス/失敗数
- 失敗したテストのエラーメッセージ
- カバレッジ（必要な場合）

### 2. 失敗時の対応

#### 軽微な失敗（自動修正可能）

以下の場合は自動修正してよい：
- 仕様書の変更に追従するだけの修正
- 明らかなタイポ・構文エラー

→ 修正後にユーザーに報告する

#### 複雑な失敗（要判断）

以下の場合は error-handling Skill に従う：
- 原因が複数考えられる
- 修正方法が複数ある
- 本番環境への影響がある

→ エラーの複雑さに応じて一問一答または質問ドキュメントで対応する

### 3. E2E テストの実行

ローカル環境:

```bash
docker compose up -d  # DynamoDB Local が起動していることを確認
ENV=local bash tests/e2e/test_api.sh
```

本番環境:

```bash
ENV=production bash tests/e2e/test_api.sh
```

### 4. テスト完了の報告

以下の形式で報告すること：

```
## テスト結果

### ユニットテスト
  合計: X件 / 成功: X件 / 失敗: X件

### E2E テスト（実行した場合）
  合計: X件 / 成功: X件 / 失敗: X件

### 失敗したテスト（ある場合）
  - テスト名: エラー内容

### 次のアクション
  （問題がある場合の推奨アクション）
```
