# Question Document Workflow

## いつ使うか

以下の場合はコンソールでの一問一答ではなく、
質問ドキュメントを作成すること：

- 複数の質問が相互に関連している場合
- 回答に前提条件の整理が必要な場合
- 全体的な方向性・設計判断を決める必要がある場合
- 選択肢が多く、一覧で見た方が判断しやすい場合
- 決定事項を記録として残す必要がある場合

## いつ使わないか

以下の場合はコンソールで直接質問してよい：

- Yes/No で答えられる単一の質問
- 明確な二択・三択の選択
- 即座に回答が得られる事実確認

## ワークフロー

### Step 1：質問ドキュメントの作成

`docs/questions/` ディレクトリに以下の形式で作成する：

ファイル名: `YYYY-MM-DD_<トピック>.md`
例: `2026-04-02_api-authentication.md`

```markdown
# 質問: <トピックの要約>

## 背景
（なぜこの判断が必要なのか、コンテキストを説明）

## 質問一覧

### Q1: <質問タイトル>
（質問の詳細説明）

選択肢:
- A. （選択肢の説明）
- B. （選択肢の説明）
- C. （選択肢の説明）

回答: <!-- ここにユーザーが回答を記入 -->

---

### Q2: <質問タイトル>
（質問の詳細説明）

※ この質問は Q1 の回答に依存します

回答: <!-- ここにユーザーが回答を記入 -->

---

## 補足メモ
（ユーザーが自由に追記するスペース）
```

### Step 2：ユーザーへの通知

質問ドキュメントを作成したら、コンソールで以下を伝える：

```
質問ドキュメントを作成しました：
  docs/questions/YYYY-MM-DD_<トピック>.md

X件の質問があります。
ドキュメント内の「回答:」欄に記入してください。
回答が完了したら「回答した」と伝えてください。
```

### Step 3：回答の読み取りと実行

ユーザーが「回答した」「回答を確認して」などと伝えたら：

1. 質問ドキュメントを読み込む
2. 全回答を確認する
3. 未回答の質問があれば指摘する
4. 回答に基づいて作業を実行する
5. 実行結果を報告する

### Step 4：決定事項の記録

作業完了後、質問ドキュメントの末尾に
決定事項と実行結果を追記する：

```markdown
## 決定事項（Claude Code が記入）
- Q1: A を選択 → （実行した内容）
- Q2: （回答内容） → （実行した内容）

## 実行日
YYYY-MM-DD
```

## 注意事項

- `docs/questions/` は .gitignore に追加しないこと
  （決定の記録として残す価値がある）
- 回答待ちの間、他の作業を進めてよい
- 緊急性が高い質問は質問ドキュメントではなく
  コンソールで直接確認すること
